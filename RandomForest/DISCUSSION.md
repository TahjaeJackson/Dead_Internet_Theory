# Detecting Bots in Reddit Activity: Exploring the Dead Internet Theory

This project investigates whether behavioural signals from Reddit activity can be used to distinguish bots from human users. The motivation comes from the Dead Internet Theory, which suggests that a large portion of online activity may already be generated by automated systems rather than real people. If this idea has any truth to it, automated behavior should leave statistical patterns that can be detected.

To explore this idea, a Random Forest classifier was trained using behavioral features extracted from Reddit user activity. These features describe patterns such as reply speed, writing style, link usage, and account characteristics. The objective is to determine whether these signals alone are sufficient for a machine learning model to identify bots.

## Dataset

The dataset contains 500 total observations.

400 rows are used for training and 100 rows are used for testing. Each row represents a Reddit user observation described by 12 behavioral features.

Examples of features include reply_delay_seconds (how quickly a user replies), avg_word_length (average word length in comments), contains_links (whether a message contains links), account_age_days (age of the account), and subreddit activity indicators such as gaming, politics, or funny.

The model learns patterns using the training set and is evaluated on the test set, which represents unseen data.

## Initial Model Results

After training the Random Forest model, the evaluation metrics were extremely high.

AUROC = 0.99999 
PR-AUC = 0.9999 
Sensitivity = 1  
Specificity = 1  
PPV = 1  
NPV = 1  

Results this strong immediately raise suspicion in machine learning. When classification performance is nearly perfect, the first concern is usually data leakage. Data leakage occurs when information from the test set accidentally influences training, making the model appear unrealistically accurate.

With this possibility, the next step was to verify carefully that the model had not accidentally seen any test data.

## Checking for Data Leakage

First, the dataset split was verified.

Training rows: 400  
Testing rows: 100  

The model was trained only on the training set.

Next, a duplicate-row check was performed by merging the training and testing feature tables and counting overlapping rows.

The overlap count returned 0.This confirms that no observations appear in both the training and testing datasets.
To further validate the model, 5-fold cross-validation was performed on the training data. The cross-validation scores were:

**[1, 1, 1, 1, 1] The mean score was therefore 1.0.**

This means that even when the training data is repeatedly split into new internal training and validation sets, the model still achieves perfect classification. This strongly suggests the high performance is not caused by memorization but by strong predictive patterns in the features.

## Why Random Forest Works Well

Random Forest is an ensemble learning algorithm that builds many decision trees and combines their predictions.
Each tree learns simple behavioural rules such as extremely fast replies indicating automated behavior, frequent links indicating promotional bots, or extremely new accounts suggesting automation.

The final prediction is determined by majority voting across trees.

$$
Prediction = majority\_vote(tree_1, tree_2, ..., tree_n)
$$

Due to many trees contributing to the decision, Random Forest reduces variance and is robust to noise. This makes it well suited for behavioral datasets with nonlinear relationships.

## ROC Curve

![ROC Curve](random_forest_roc.png)

The ROC curve compares the true positive rate and false positive rate.

The true positive rate is defined as

$$
TPR = \frac{TP}{TP + FN}
$$

The false positive rate is defined as

$$
FPR = \frac{FP}{FP + TN}
$$

The area under this curve measures how well the classifier separates bots from humans.

$$
AUC = \int_0^1 TPR(FPR) dFPR
$$

The model achieves an AUC extremely close to 1, meaning the classifier almost perfectly separates the two classes.

## Precision-Recall Curve

![Precision Recall Curve](random_forest_pr.png)

The precision-recall curve focuses on prediction accuracy for the positive class.

Precision is defined as

$$
Precision = \frac{TP}{TP + FP}
$$

Recall is defined as

$$
Recall = \frac{TP}{TP + FN}
$$

The PR-AUC is essentially 1, meaning that when the model predicts a bot, it is almost always correct.

## Threshold Tradeoff

![Threshold Tradeoff](random_forest_threshold_tradeoff.png)

Machine learning models produce probabilities rather than direct classifications. A threshold converts probabilities into labels.

For example

$$
P(bot) > 0.5 \rightarrow bot
$$

Sensitivity and specificity change as this threshold moves.

Sensitivity is

$$
Sensitivity = \frac{TP}{TP + FN}
$$

Specificity is

$$
Specificity = \frac{TN}{TN + FP}
$$

The plot shows that both values remain extremely high across thresholds, indicating that predictions are stable.

## Calibration Curve

![Calibration Curve](random_forest_calibration.png)

Calibration measures whether predicted probabilities correspond to actual outcomes.

Perfect calibration follows

$$
P(predicted) = P(observed)
$$

The calibration curve shows predictions clustering near 0 and 1, meaning the model is very confident when making predictions.

## Decision Curve

![Decision Curve](random_forest_decision_curve.png)

Decision curve analysis measures the net benefit of using the model.

$$
NetBenefit = \frac{TP}{N} - \frac{FP}{N}\frac{p}{1-p}
$$

where p is the classification threshold.

The curve shows the model consistently produces positive net benefit compared with ignoring predictions.

Random Forest also calculates feature importance, which measures how much each feature contributes to reducing classification error.

These results indicate that behavioural timing and text structure are the strongest signals for detecting bots.

## Final Interpretation and Conclusion

Bots in this dataset appear to behave differently from human users in several ways. They reply much faster, frequently include links, tend to produce longer structured text, and often originate from newer accounts.
These strong behavioural signals create a clear separation between classes, allowing the model to achieve extremely high performance.

### Connection to the Dead Internet Theory

The Dead Internet Theory proposes that much of online content may already be produced by automated systems. This experiment shows that automated behavior leaves detectable statistical patterns. Using only a small set of behavioral features, the Random Forrest machine learning model was able to identify bots with near-perfect accuracy. This suggests that, at least for this dataset, bots still behave differently enough from humans that their activity can be detected algorithmically.

### Conclusion

This project trained a Random Forest classifier to detect bots using behavioral Reddit features. Initially, the extremely high AUROC and PR-AUC values raised concerns about possible data leakage. Multiple checks were performed, including dataset separation verification, duplicate-row detection, and cross-validation. These checks confirmed that the high performance is genuine and results from strong predictive patterns in the data. The results demonstrate that behavioural features such as reply timing, word structure, and link usage can strongly distinguish automated accounts from human users. This supports the idea that machine learning models can play an important role in identifying automated activity on the internet.
